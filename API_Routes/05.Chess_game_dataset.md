# ♟️ Chess Match Analytics API Documentation
---

## 🎮 Match Management & CRUD

*Core operations for handling match records and historical data.*

* `GET /api/v1/matches` - Fetch a paginated list of all recorded chess matches.
* `GET /api/v1/matches/:id` - Retrieve full details of a specific match (includes full move string).
* `POST /api/v1/matches` - Log a new match result into the database.
* `PUT /api/v1/matches/:id` - Replace an existing match record.
* `PATCH /api/v1/matches/:id/status` - Update specific details like `victory_status`.
* `DELETE /api/v1/matches/:id` - Remove a match record from history.
* `GET /api/v1/matches/:id/moves` - Fetch only the `moves` string for a specific game.

## 👤 Player & Ranking Routes

*Routes focused on individual performance and ELO ratings.*

* `GET /api/v1/players/:username/history` - Get all matches played by a specific user (White or Black).
* `GET /api/v1/players/:username/stats` - Get win/loss/draw ratio and average rating for a player.
* `GET /api/v1/players/top-rated` - List players with the highest ELO ratings in the dataset.
* `GET /api/v1/players/compare?p1=user1&p2=user2` - Compare head-to-head stats between two players.
* `GET /api/v1/players/rating-range?min=1500&max=2000` - Filter matches where both players fall within a rating bracket.
* `GET /api/v1/players/:username/openings` - Identify the most frequent openings used by a specific player.

---

## 📖 Opening Theory & Analysis

*Routes for studying the "Opening Phase" of the game.*

* `GET /api/v1/openings` - List all unique openings (e.g., "Sicilian Defense") found in the data.
* `GET /api/v1/openings/popular` - Get the top 10 most played openings.
* `GET /api/v1/openings/win-rates` - Calculate which openings have the highest win percentage for White.
* `GET /api/v1/openings/eco/:code` - Filter games by ECO (Encyclopaedia of Chess Openings) code.
* `GET /api/v1/openings/search?name=Gambit` - Search for specific opening names.
* `GET /api/v1/openings/complexity?min_ply=5` - Filter games based on the `opening_ply` depth.

## 📊 Match Analytics & Trends

*Routes focused on the mechanics of the games played.*

* `GET /api/v1/analytics/victory-distribution` - Count games ended by Resignation, Mate, Out of Time, or Draw.
* `GET /api/v1/analytics/color-advantage` - Compare total wins for White vs. Black.
* `GET /api/v1/analytics/turn-count-avg` - Calculate the average number of turns per game.
* `GET /api/v1/analytics/rated-vs-casual` - Statistics comparing rated matches vs. unrated matches.
* `GET /api/v1/analytics/time-control-usage` - Distribution of different `increment_code` types (e.g., 15+2, 10+0).
* `GET /api/v1/analytics/shortest-games` - Retrieve matches that ended in the fewest turns.
* `GET /api/v1/analytics/rating-gap-upsets` - Find games where a significantly lower-rated player beat a higher-rated one.

## 🚀 Advanced Features & Bulk Actions

*Simulating advanced game-platform functionality.*

* `GET /api/v1/matches/filter?rated=true&winner=black&min_turns=20` - Multi-parameter filter.
* `POST /api/v1/matches/bulk-upload` - Upload an array of game records from a JSON file.
* `GET /api/v1/matches/export/pgn/:id` - Export a match in standard PGN (Portable Game Notation) format.
* `GET /api/v1/matches/discover/random` - Fetch a random high-rated match for analysis.
* `GET /api/v1/matches/trending-openings` - Openings that have seen an increase in usage recently.
* `GET /api/v1/matches/active-hours` - Analyze `created_at` timestamps to find peak playing times.
* `DELETE /api/v1/matches/cleanup/unrated` - Admin route to purge unrated games for data cleaning.
* `GET /api/v1/matches/search/moves?sequence=e4,e5` - Find games that started with a specific move sequence.

*Backend health and developer tools.*

* `GET /api/v1/system/logs` - Retrieve recent match-processing logs.

## ✨ Good to Have (Chess Specific Enhancements)

* **ELO Logic:** Practice writing a function that calculates expected win probability based on the `white_rating` and `black_rating`.
* **Move Parser:** The `moves` field is a long string. Practice writing a parser that breaks this string into an array of "Move Objects" (e.g., `[{turn: 1, white: "d4", black: "d5"}]`).
* **Validation:** Before allowing a `POST` request, validate that the `increment_code` follows the "Minutes+Seconds" format.
* **Timestamp Conversion:** The `created_at` field is in milliseconds (Unix timestamp). Practice converting this into human-readable dates (e.g., `May 11, 2026`).
* **Board Visualization:** Link matches to a frontend library like `Chessboard.js` to replay the moves stored in the API.
